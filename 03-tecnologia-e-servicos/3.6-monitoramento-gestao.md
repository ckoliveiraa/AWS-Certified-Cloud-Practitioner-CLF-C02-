## Amazon CloudWatch — recursos avançados

CloudWatch é o **olho da AWS sobre tudo**. Métricas, logs, alarmes e dashboards já vimos na 2.7. Aqui ficam os recursos extras:

| Recurso | Para que serve |
|---------|----------------|
| **CloudWatch Agent** | Coleta métricas **dentro** da EC2 (CPU é fácil, mas memória/disco precisa do agente) |
| **CloudWatch Logs Insights** | Linguagem de query para analisar logs (parecido com SQL) |
| **CloudWatch Synthetics** | **Canários** — robôs que testam URLs continuamente |
| **Container Insights** | Métricas detalhadas de ECS, EKS e Fargate |
| **Lambda Insights** | Métricas detalhadas de funções Lambda |
| **Composite Alarms** | Alarme que combina vários alarmes (E/OU) |

> 💡 **Pegadinha clássica:** "memória de EC2 não aparece no CloudWatch" → precisa instalar o **CloudWatch Agent**.

## AWS Health Dashboard

Mostra eventos que afetam serviços AWS.

| Tipo | O que mostra |
|------|--------------|
| **Service Health Dashboard** | Status **público** de todos os serviços AWS (visível para qualquer um) |
| **Personal Health Dashboard** | Eventos que afetam **sua conta** especificamente (ex: manutenção em sua EC2) |

> 💡 Personal Health integra com **EventBridge** — você pode automatizar respostas a problemas na sua conta.

# Infrastructure as Code (IaC)

**O que é:** descrever a infraestrutura em **arquivos de texto** versionáveis, em vez de criar tudo manualmente no console.

**Vantagens:**
- Versionamento (Git)
- Reprodutibilidade (mesmo ambiente em dev/prod)
- Automação (CI/CD)
- Documentação viva

## AWS CloudFormation

**O que é:** o serviço nativo de IaC da AWS, em **JSON ou YAML**.

### Conceitos principais

| Conceito | O que é |
|----------|---------|
| **Template** | Arquivo JSON/YAML que descreve recursos |
| **Stack** | Conjunto de recursos criados a partir de um template |
| **Change Set** | "Preview" das mudanças antes de aplicar |
| **Stack Set** | Stack replicada em **múltiplas contas e regiões** |
| **Drift Detection** | Detecta mudanças feitas manualmente (fora do template) |
| **Rollback** | Reverte automaticamente em caso de falha |

### Características que caem na prova

- ✅ **Gratuito** — paga só pelos recursos criados
- ✅ Suporta **rollback automático** quando algo falha
- ✅ **Drift Detection** alerta sobre mudanças manuais
- ✅ **Stack Sets** = replicação multi-conta/multi-região (ideal com Organizations)

> 💡 **Pegadinha:** alguém alterou um SG manualmente no console → CloudFormation **não sabe**. Use **Drift Detection** para descobrir.

## AWS CDK (Cloud Development Kit)

**O que é:** IaC em **linguagem de programação** (TypeScript, Python, Java, C#, Go).

### Por que existe (se já tem CloudFormation)?

CloudFormation YAML fica **enorme e repetitivo**. CDK permite usar **loops, condicionais e abstrações**:

```typescript
// CDK: 3 linhas
for (const env of ['dev', 'staging', 'prod']) {
  new Bucket(this, `bucket-${env}`);
}
// CloudFormation equivalente: 90+ linhas de YAML
```

> 💡 CDK **não substitui** CloudFormation — ele **gera** templates CloudFormation por baixo.

| | CloudFormation | CDK |
|---|---|---|
| Linguagem | YAML/JSON | TypeScript, Python, Java... |
| Curva de aprendizado | Média | Baixa para devs |
| Reutilização | Limitada | Alta (constructs) |
| Quem prefere | DevOps tradicional | Devs |

# Operação e Gestão de Frota

## AWS Systems Manager (SSM)

**O que é:** suíte de ferramentas para **operar** EC2 e on-premises em escala.

**Analogia:** se EC2 é uma frota de carros, Systems Manager é o **sistema de gestão da frota**.

### Os 6 recursos que importam

| Recurso | Para que serve |
|---------|----------------|
| **Session Manager** | Acesso shell **sem SSH/RDP** (sem porta 22 aberta!) |
| **Run Command** | Executa comando em **dezenas de instâncias** ao mesmo tempo |
| **Patch Manager** | Aplica patches em massa, com janelas de manutenção |
| **Parameter Store** | Guarda **configs e segredos simples** (gratuito) — comparação completa com Secrets Manager em [2.4](../02-seguranca-e-conformidade/2.4-criptografia.md) |
| **State Manager** | Mantém configuração **desejada** (auto-corrige drift) |
| **Inventory** | Inventário de **software instalado** em cada instância |

---

## AWS OpsWorks

- Gerencia **Chef** e **Puppet** na AWS.
- Caindo em desuso — para a prova, basta saber que é "AWS + Chef/Puppet".

## AWS Service Catalog

- **Catálogo de produtos aprovados** para self-service em empresas.
- Usuários criam recursos pré-aprovados sem mexer em console AWS.
- Útil para empresas com **governança rígida**.

## AWS License Manager

- Gerencia licenças de software (**Oracle, SQL Server, SAP, Windows**).
- Evita estouro de licenças (gera alertas).

## AWS Compute Optimizer

- Analisa uso de **EC2, EBS, Lambda, ASG** e recomenda **instâncias otimizadas**.
- Detecta **over-provisioning** (paga mais do que precisa).
- **Gratuito**.

> 💡 **Cenário típico:** "empresa quer reduzir custos identificando recursos super-dimensionados" → **Compute Optimizer**.

## Tabela de cenários típicos da prova

| Cenário | Serviço |
|---------|---------|
| "Acessar EC2 sem abrir porta 22 (SSH)" | **Session Manager** |

| "Detectar mudança manual fora do template" | **CloudFormation Drift Detection** |
| "Replicar template em várias contas/regiões" | **Stack Sets** |
| "IaC com loops e condicionais" | **CDK** |
| "Memória de EC2 no CloudWatch" | **CloudWatch Agent** |
| "Testar URL continuamente de várias regiões" | **CloudWatch Synthetics** |
| "Reduzir custo identificando EC2 super-dimensionada" | **Compute Optimizer** |
| "Catálogo de produtos pré-aprovados em empresa" | **Service Catalog** |
| "Gerenciar licenças Oracle/SQL Server" | **License Manager** |
| "Eventos que afetam minha conta especificamente" | **Personal Health Dashboard** |

## Pontos-Chave para o Exame

- ✅ **CloudFormation** = IaC nativo em **JSON/YAML**, **gratuito**, suporta **Drift Detection** e **Stack Sets**.
- ✅ **CDK** = IaC em **código** (TS/Python/Java), gera CloudFormation por baixo.
- ✅ **Session Manager** elimina necessidade de SSH/RDP.
- ✅ **Parameter Store** (grátis) e **Secrets Manager** (rotação automática) — detalhes em [Aula 2.4](../02-seguranca-e-conformidade/2.4-criptografia.md).
- ✅ **CloudWatch Agent** é necessário para métricas que **vivem dentro do SO** (memória, disco).
- ✅ **Compute Optimizer** recomenda instâncias certas (reduz custo).
- ✅ **Personal Health Dashboard** = só sua conta · **Service Health** = público.
- ✅ **Trusted Advisor com tier completo** exige plano **Business** ou **Enterprise**.
