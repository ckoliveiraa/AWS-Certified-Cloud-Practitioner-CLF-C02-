# 3.7 — Integração e Mensageria

## Amazon SNS (Simple Notification Service)

- **Pub/Sub** — um publisher → muitos subscribers.
- Subscribers: **SQS, Lambda, HTTP, Email, SMS, mobile push**.
- **Topic-based**.

## Amazon SQS (Simple Queue Service)

- **Fila** de mensagens (pull).
- Tipos:
  - **Standard** — throughput alto, entrega at-least-once.
  - **FIFO** — ordem garantida, exactly-once.
- Mensagens até **256 KB**, retenção até **14 dias**.

## Amazon EventBridge (antigo CloudWatch Events)

- **Event bus** serverless.
- Roteia eventos entre serviços AWS, SaaS e apps.
- **Schema Registry** e regras baseadas em padrões.

## AWS Step Functions

- **Orquestração** de workflows (state machines).
- Integra Lambda, ECS, SNS, DynamoDB…
- Visual workflow JSON.

## Amazon MQ

- **Message broker gerenciado** (ActiveMQ, RabbitMQ).
- Para sistemas legados que usam protocolos padrão (AMQP, MQTT, STOMP).

## Amazon AppFlow

- Integração **SaaS ↔ AWS** sem código (Salesforce, ServiceNow…).

---

## Comparação

| Serviço | Padrão | Ordem |
|---------|--------|-------|
| **SNS** | Pub/Sub (push) | Não garante |
| **SQS** | Fila (pull) | FIFO opcional |
| **EventBridge** | Event bus (pattern matching) | Não |
| **Step Functions** | Orquestração de fluxos | Determinística |
| **MQ** | Broker padrão (AMQP, MQTT) | Sim |

---

## Pontos-Chave para o Exame

- ✅ **SNS = push**, **SQS = pull**.
- ✅ **SQS FIFO** para ordem garantida.
- ✅ **EventBridge** é evolução do CloudWatch Events com integração SaaS.
- ✅ **Step Functions** para workflows visuais.
- ✅ **Amazon MQ** para sistemas legados com protocolos padrão.
