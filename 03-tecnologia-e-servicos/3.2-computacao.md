# 3.2 — Computação

## Amazon EC2 (Elastic Compute Cloud)

- **Máquinas virtuais** on-demand na AWS.
- **Famílias de instância**: General (T, M), Compute (C), Memory (R, X), Storage (I, D), GPU (P, G).
- **AMI** (Amazon Machine Image) = template da instância.
- **EC2 Image Builder** — pipeline de criação de AMIs.

### Modelos de Preço
| Modelo | Quando usar |
|--------|-------------|
| **On-Demand** | Workloads curtos/imprevisíveis |
| **Reserved (RI)** | Uso previsível 1–3 anos (até 72% off) |
| **Savings Plans** | Compromisso $/h flexível (cobre EC2, Fargate, Lambda) |
| **Spot** | Workloads tolerantes a falhas (até 90% off, aviso de 2 min) |
| **Capacity Reservations** | Reserva capacidade em uma AZ sem compromisso de tempo |
| **Dedicated Host** | Conformidade/licenciamento BYOL (Oracle, SQL Server) |
| **Dedicated Instance** | Hardware dedicado (sem visibilidade do host) |

> ⚠️ **Spot NÃO serve para:** banco de dados de produção, web crítico. **Serve para:** batch, CI/CD, render, ML training.

### EC2 Auto Scaling
- Escala instâncias automaticamente conforme demanda. Trabalha com **ELB** para HA + elasticidade.
- **Launch Template** — configuração da instância (AMI, tipo, SG).
- **Auto Scaling Group (ASG)** — define min/max/desejado, distribui em múltiplas AZs.
- **Políticas:** Target Tracking (mantém métrica), Step Scaling (degraus), Scheduled (horários).

## AWS Lambda

- **Serverless** — executa código em resposta a eventos.
- Paga por **execução + duração** (em ms).
- Linguagens: Python, Node.js, Java, Go, Ruby, .NET, custom runtimes.

### Limites importantes
| Limite | Valor |
|---|---|
| Timeout máximo | **15 minutos** |
| Memória | 128 MB a 10 GB (CPU proporcional) |
| Pacote zipado | 50 MB (250 MB descompactado) |
| Container image | até 10 GB |

- **Cold start** — primeira invocação após ociosidade demora mais. Mitigação: **Provisioned Concurrency**.
- **Free tier:** 1M requisições + 400.000 GB-segundos/mês — **para sempre**.

## Containers na AWS

| Serviço | Descrição |
|---------|-----------|
| **Amazon ECS** | Orquestração de containers (AWS nativo) |
| **Amazon EKS** | Kubernetes gerenciado |
| **AWS Fargate** | Serverless para containers (ECS/EKS) |
| **Amazon ECR** | Registro de imagens Docker |

---

## AWS Elastic Beanstalk

- **PaaS** — envie o código, a AWS provisiona tudo.
- Suporta: Java, .NET, Node.js, PHP, Python, Ruby, Go, Docker.
- **Grátis** — paga apenas pelos recursos criados (EC2, ELB…).

---

## Amazon Lightsail

- **VPS simplificado** com preço fixo mensal.
- Ideal para sites pequenos, blogs, devs iniciantes.

---

## AWS Batch

- Processamento **em lote** em larga escala.
- Gerencia filas e recursos (EC2/Fargate) automaticamente.

---

## AWS Outposts & Wavelength *(revisão)*

- **Outposts** — AWS no seu DC (híbrido).
- **Wavelength** — AWS em redes 5G.

---

## Quando usar cada serviço de computação

| Cenário | Serviço |
|---|---|
| Controle total do SO, app legado | **EC2** |
| Evento curto (<15 min), código pequeno | **Lambda** |
| App em container, controle do cluster | **ECS/EKS no EC2** |
| Container sem gerenciar servidor | **Fargate** |
| Desenvolvedor que só quer enviar o código | **Beanstalk** |
| Site/blog simples, preço fixo | **Lightsail** |
| Processamento em lote massivo | **AWS Batch** |

> 💡 **Analogia:** EC2 = alugar um carro · Lambda = Uber · Beanstalk = táxi · Containers = van padronizada.

---

## Free Tier — resumo

- **EC2:** 750h/mês de t2.micro ou t3.micro por 12 meses.
- **Lambda:** 1M requisições + 400.000 GB-s/mês — **para sempre**.
- **Lightsail:** 3 meses grátis no plano de US$ 3,50.

---

## Pontos-Chave para o Exame

- ✅ **EC2 = IaaS**, **Lambda = serverless**, **Beanstalk = PaaS**.
- ✅ **Spot** até 90% off (tolerante a falhas, aviso de 2 min).
- ✅ **Reserved (1 ou 3 anos)** — melhor para uso previsível.
- ✅ **Savings Plans** cobre EC2 + Fargate + Lambda.
- ✅ **Fargate** = containers sem gerenciar servidor.
- ✅ Lambda limitado a **15 min de execução**, memória 128 MB–10 GB.
- ✅ **Auto Scaling + ELB** = alta disponibilidade + elasticidade.
- ✅ **Dedicated Host** é para BYOL; **Dedicated Instance** é hardware dedicado sem visibilidade.
