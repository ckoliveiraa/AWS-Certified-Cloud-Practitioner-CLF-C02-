# 3.4 — Bancos de Dados

## Amazon RDS

- **Relacional gerenciado** — AWS cuida de backup, patch, HA.
- Engines: **MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, Db2, Aurora**.
- **Multi-AZ** — standby síncrono para failover (HA).
- **Read Replicas** — escalabilidade de leitura (assíncrono).

---

## Amazon Aurora

- Compatível com **MySQL e PostgreSQL**.
- **3x–5x mais rápido** que os equivalentes.
- **Armazenamento auto-escalável** até 128 TB.
- **6 cópias em 3 AZs** por padrão.
- **Aurora Serverless** — auto-scale e paga por uso.

---

## Amazon DynamoDB

- **NoSQL key-value** serverless.
- **Latência < 10ms** em single-digit ms.
- Escala para **milhões de req/s**.
- **Global Tables** — multi-região ativo-ativo.
- **DAX** — cache em memória (microsegundos).

---

## Amazon Redshift

- **Data warehouse** (OLAP).
- **PostgreSQL-like**, colunar.
- Petabytes em escala.
- **Redshift Serverless** disponível.

---

## Amazon ElastiCache

- Cache gerenciado.
- Engines: **Redis** e **Memcached**.
- Reduz carga de BD relacional.

---

## Outros Bancos Gerenciados

| Serviço | Tipo | Caso de uso |
|---------|------|-------------|
| **DocumentDB** | NoSQL (MongoDB-compat.) | Documentos JSON |
| **Neptune** | Grafo | Redes sociais, fraude |
| **Timestream** | Séries temporais | IoT, métricas |
| **QLDB** | Ledger imutável | Auditoria |
| **Keyspaces** | Cassandra-compat. | Wide-column |
| **MemoryDB for Redis** | In-memory com durabilidade | Cache durável |

---

## AWS DMS (Database Migration Service)

- Migração entre bancos (homogênea ou heterogênea).
- **SCT** (Schema Conversion Tool) para conversão de engine.

---

## Pontos-Chave para o Exame

- ✅ **RDS = relacional gerenciado**; **Multi-AZ = HA**, **Read Replica = performance de leitura**.
- ✅ **Aurora** = 5x MySQL / 3x PostgreSQL.
- ✅ **DynamoDB = NoSQL serverless**, low latency.
- ✅ **Redshift = data warehouse** (OLAP, não OLTP).
- ✅ **ElastiCache** = Redis ou Memcached.
