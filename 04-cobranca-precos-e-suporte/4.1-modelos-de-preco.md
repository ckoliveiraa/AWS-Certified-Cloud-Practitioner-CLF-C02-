# 4.1 — Modelos de Preço AWS

## Os 3 Princípios de Precificação AWS

1. **Pay as you go** — pague pelo que usar, sem compromisso.
2. **Save when you reserve** — comprometa-se (RI / Savings Plans) e ganhe desconto.
3. **Pay less by using more** — descontos por volume (ex.: S3 com tiers de preço por GB).

## O que a AWS cobra

1. **Computação** — tempo e tipo de instância.
2. **Armazenamento** — GB armazenados.
3. **Transferência de dados** — **saída** geralmente é cobrada; **entrada é gratuita**.

> ⚠ **Preço varia por Região** — a mesma instância pode custar diferente em us-east-1 vs sa-east-1.

### Transferência de dados — detalhe (cai na prova)

| Cenário | Custo |
|---------|-------|
| Entrada (inbound) para AWS | **Grátis** |
| Saída para internet | Cobrado |
| Entre serviços na **mesma AZ** (IP privado) | Geralmente grátis |
| Entre **AZs diferentes** na mesma Região | Cobrado |
| Entre **Regiões** | Cobrado (mais caro) |

## Modelos de Compra EC2

| Modelo | Compromisso | Desconto | Uso ideal |
|--------|-------------|----------|-----------|
| **On-Demand** | Nenhum | 0% | Curto prazo, imprevisível |
| **Reserved Instance (1 ou 3 anos)** | Alto | Até **72%** | Uso previsível |
| **Savings Plans** | 1 ou 3 anos, $/hora | Até **72%** | Flexível |
| **Spot** | Nenhum | Até **90%** | Batch, big data, CI/CD, workloads stateless/tolerantes a falhas |
| **Dedicated Host** | Físico dedicado | — | **BYOL** com afinidade de socket/core, conformidade |
| **Dedicated Instance** | Hardware dedicado | — | Sem visibilidade do host físico |
| **Capacity Reservation** | Capacidade garantida | **Sem desconto** (cobra On-Demand) | Garantir provisão em uma AZ |

### Reserved Instances — detalhes
- **Standard** — maior desconto, menos flexível.
- **Convertible** — pode mudar família/SO.
- Pagamento: **All Upfront, Partial Upfront, No Upfront**.

### Savings Plans
- **Compute Savings Plan** — mais flexível (EC2 + Fargate + Lambda).
- **EC2 Instance Savings Plan** — maior desconto, preso a família/região.
- ⚠ **Não cobre RDS, DynamoDB, ElastiCache, Redshift** — esses têm seus próprios Reserved Instances/Nodes.

## Free Tier

Três categorias:
| Tipo | Exemplos |
|------|----------|
| **12 meses grátis** | 750h EC2 t2/t3.micro, 5 GB S3, 750h RDS |
| **Sempre grátis** | 1M req/mês Lambda, 25 GB DynamoDB |
| **Trials** | SageMaker (2 meses), Redshift (2 meses) |

## AWS Pricing Calculator

- Estima custo **antes** de provisionar.
- **Gratuito**, sem login obrigatório.

## TCO (Total Cost of Ownership)

- Compara custo **on-premises vs nuvem**.
- Considera: hardware, energia, equipe, licenças, espaço físico.

---

## Pontos-Chave para o Exame

- ✅ **Entrada de dados = grátis**; saída geralmente cobrada.
- ✅ **Spot = até 90% off** (mas interruptível).
- ✅ **Reserved / Savings Plans = até 72% off**, 1 ou 3 anos.
- ✅ **Pricing Calculator** estima; **TCO** compara com on-premises.
- ✅ **Free Tier** tem **3 tipos**: *12-Month Free*, *Always Free*, *Trials*.
- ✅ **Savings Plans não cobrem RDS/DynamoDB** — esses usam RI próprio.
- ✅ **Capacity Reservation não dá desconto** — só garante capacidade.
- ✅ **Preço varia por Região**; transferência entre AZs/Regiões é cobrada.
