# 2.7 — Auditoria e Conformidade

## AWS CloudTrail

- Registra **todas as chamadas de API** na conta AWS.
- Responde a: *"Quem fez o quê, quando e de onde?"*
- **Habilitado por padrão** (últimos 90 dias visíveis).
- Para retenção longa → enviar para **S3**.
- Alertas → integra com **CloudWatch Events / EventBridge**.

## Amazon CloudWatch

- **Monitoramento** de métricas, logs e eventos.
- **Métricas** — CPU, memória, rede.
- **Logs** — logs de aplicações.
- **Alarms** — disparam ações (SNS, Auto Scaling).
- **Dashboards** customizáveis.

## AWS Config

- Registra **configurações** dos recursos AWS ao longo do tempo.
- Responde a: *"Como estava esse recurso ontem às 14h?"*
- **Regras** (managed e custom) avaliam conformidade.
- Integra com **Security Hub**.

## AWS Artifact

- Portal de **relatórios de conformidade** (SOC, ISO, PCI-DSS, etc.).
- Usado para **auditorias externas**.
- **Gratuito**.

## AWS Audit Manager

- Automatiza **coleta de evidências** para auditorias.
- Frameworks prontos: CIS, GDPR, HIPAA, PCI-DSS.

## Comparação — Quem Faz o Quê?

| Pergunta | Serviço | Tipo de dado | Resposta em… |
|----------|---------|--------------|--------------|
| **Quem** chamou essa API? | **CloudTrail** | Eventos de API (JSON) | Segundos a minutos |
| Esse SG foi modificado? **Quando**? | **CloudTrail** | Histórico de chamadas | Histórico (90 dias padrão) |
| A **CPU/memória** está alta? | **CloudWatch Metrics** | Séries temporais numéricas | Tempo real |
| Cadê os **logs da minha aplicação**? | **CloudWatch Logs** | Texto/JSON de apps | Tempo real |
| **Como está** configurado esse recurso **agora**? | **AWS Config** | Snapshot de configuração | Sob demanda |
| **O que mudou** na configuração desde ontem? | **AWS Config** | Histórico de configuração | Linha do tempo |
| Esse recurso está **conforme** a regra X? | **AWS Config Rules** | Avaliação de compliance | Contínuo |
| Preciso do **relatório SOC 2 / PCI** da AWS | **AWS Artifact** | PDF de certificação | Download imediato |
| **Automatizar** coleta de evidências PCI/HIPAA | **Audit Manager** | Pacote de auditoria | Contínuo |

### Pegadinha clássica: CloudTrail vs Config

| Pergunta | Resposta |
|----------|----------|
| **Quem** modificou? | **CloudTrail** |
| **O que** foi modificado? | **CloudTrail** (parcial) + **Config** (estado) |
| **Como estava antes** vs **agora**? | **Config** (linha do tempo de configuração) |
| **Quando** mudou? | Ambos — CloudTrail por evento, Config por snapshot |

---

## Pontos-Chave para o Exame

- ✅ **CloudTrail = API calls**; **CloudWatch = métricas/logs**; **Config = mudanças de configuração**.
- ✅ **Artifact = baixar relatórios** de conformidade (SOC, ISO, PCI).
- ✅ CloudTrail **não** é para logs de aplicação (isso é CloudWatch Logs).
- ✅ Config responde **"o que mudou?"**; CloudTrail responde **"quem mudou?"**.
