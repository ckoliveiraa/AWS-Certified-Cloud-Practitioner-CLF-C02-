# 2.6 — Detecção e Resposta a Incidentes

| Serviço | Função | Fonte de dados |
|---------|--------|----------------|
| **GuardDuty** | Detecção de **ameaças** com ML | CloudTrail, VPC Flow Logs, DNS logs |
| **Inspector** | Avaliação de **vulnerabilidades** | EC2, ECR (imagens), Lambda |
| **Macie** | Descoberta de **dados sensíveis** (PII) | **S3** |
| **Security Hub** | **Dashboard central** de segurança | Agrega GuardDuty, Inspector, Macie, Config |
| **Detective** | **Investigação** de causa raiz | Logs + grafos |
| **Trusted Advisor** | Recomendações de segurança, custo, performance | Configuração da conta |

---

## Amazon GuardDuty

- Detecta **comportamentos anômalos**: instâncias mineração cripto, chamadas de API suspeitas, exfiltração.
- **Sem agentes** — usa logs.
- Ativação com 1 clique.

## Amazon Inspector

- Escaneia **CVEs** em EC2, imagens ECR e funções Lambda.
- Pontuação de risco.

## Amazon Macie

- Usa **ML** para identificar **PII, PHI, credenciais** em **S3**.
- Alerta para buckets expostos.

## AWS Security Hub

- Consolida achados de vários serviços de segurança.
- Verifica conformidade contra: **CIS, PCI-DSS, AWS Foundational Best Practices**.

## Amazon Detective

- Recebe eventos do GuardDuty e **visualiza relações** entre entidades.
- Acelera investigação forense.

---

## Fluxo Típico de Resposta

```
GuardDuty detecta → Security Hub agrega → Detective investiga → Resposta manual/SOAR
```

---

## Pontos-Chave para o Exame

- ✅ **GuardDuty** = ameaças; **Inspector** = vulnerabilidades; **Macie** = dados sensíveis no S3.
- ✅ **Security Hub** é o **dashboard central**.
- ✅ **Detective** = investigação (não detecção).
- ✅ Todos são **regionais** (exceto Security Hub multi-região).
