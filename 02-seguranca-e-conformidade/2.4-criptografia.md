# 2.4 — Criptografia: KMS, CloudHSM, ACM, Secrets Manager

## AWS KMS — Key Management Service

- Cria e gerencia **chaves criptográficas** (CMKs / KMS Keys).
- Integrado com **maioria dos serviços AWS** (S3, EBS, RDS, Lambda…).
- **Regional** (chave existe em uma região).
- Tipos de chaves:
  - **AWS Managed** — criadas pela AWS (ex.: `aws/s3`).
  - **Customer Managed** — você controla rotação e políticas.
  - **AWS Owned** — invisíveis para você.

---

## AWS CloudHSM

- **Hardware Security Module dedicado** (single-tenant).
- Você controla **100%** das chaves (AWS não tem acesso).
- Atende conformidade **FIPS 140-2 Level 3**.
- Quando usar? Requisitos regulatórios rigorosos.

| Recurso | KMS | CloudHSM |
|---------|-----|----------|
| Multi-tenant | ✅ | ❌ (dedicado) |
| FIPS | Level 2 | **Level 3** |
| AWS tem acesso | Sim (compartilhado) | **Não** |
| Preço | Mais barato | Mais caro |

---

## AWS Certificate Manager (ACM)

- Provisiona, gerencia e **renova gratuitamente** certificados **SSL/TLS**.
- Integrado com **ELB**, **CloudFront**, **API Gateway**.
- Não emite certificados para servidores fora da AWS.

---

## AWS Secrets Manager

- Armazena **segredos** (senhas, API keys, strings de conexão).
- **Rotação automática** (ex.: credenciais RDS a cada 30 dias).
- Preço **por segredo + chamadas API**.

## AWS Systems Manager Parameter Store

- Alternativa **gratuita** para valores de configuração.
- Suporta **SecureString** (criptografia via KMS).
- **Sem rotação automática nativa**.

| Recurso | Secrets Manager | Parameter Store |
|---------|-----------------|-----------------|
| Preço | Pago | **Grátis** (tier padrão) |
| Rotação automática | ✅ | ❌ |
| Uso ideal | Segredos sensíveis | Config geral |

---

## Pontos-Chave para o Exame

- ✅ **KMS** = chaves gerenciadas; **CloudHSM** = HSM dedicado (FIPS L3).
- ✅ **ACM** emite certificados **SSL/TLS gratuitos**.
- ✅ **Secrets Manager** = rotação automática; **Parameter Store** = grátis, sem rotação.
- ✅ Criptografia em trânsito = TLS; em repouso = KMS/SSE.
