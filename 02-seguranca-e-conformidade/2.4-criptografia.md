# 2.4 — Criptografia: KMS, CloudHSM, ACM, Secrets Manager

## AWS KMS — Key Management Service

- Cria e gerencia **chaves criptográficas** (CMKs / KMS Keys).
- Integrado com **maioria dos serviços AWS** (S3, EBS, RDS, Lambda…).
- **Regional** (chave existe em uma região).
- Tipos de chaves:
  - **AWS Managed** — criadas pela AWS (ex.: `aws/s3`).
  - **Customer Managed** — você controla rotação e políticas.
  - **AWS Owned** — invisíveis para você.

---

## AWS CloudHSM

- **Hardware Security Module dedicado** (single-tenant).
- Você controla **100%** das chaves (AWS não tem acesso).
- Atende conformidade **FIPS 140-2 Level 3**.
- Quando usar? Requisitos regulatórios rigorosos.

| Recurso | KMS | CloudHSM |
|---------|-----|----------|
| Multi-tenant | ✅ | ❌ (dedicado) |
| FIPS | Level 2 | **Level 3** |
| AWS tem acesso | Sim (compartilhado) | **Não** |
| Preço | Mais barato | Mais caro |

---

## AWS Certificate Manager (ACM)

- Provisiona, gerencia e **renova gratuitamente** certificados **SSL/TLS**.
- Integrado com **ELB**, **CloudFront**, **API Gateway**.
- Não emite certificados para servidores fora da AWS.

---

## AWS Secrets Manager

- Armazena **segredos** (senhas, API keys, strings de conexão).
- **Rotação automática** (ex.: credenciais RDS a cada 30 dias).
- Preço **por segredo + chamadas API**.

## AWS Systems Manager Parameter Store

- Alternativa **gratuita** para valores de configuração.
- Suporta **SecureString** (criptografia via KMS).
- **Sem rotação automática nativa**.

| Recurso | Secrets Manager | Parameter Store |
|---------|-----------------|-----------------|
| Preço | Pago | **Grátis** (tier padrão) |
| Rotação automática | ✅ | ❌ |
| Uso ideal | Segredos sensíveis | Config geral |

---

## Pontos-Chave para o Exame

- ✅ **KMS** = chaves gerenciadas; **CloudHSM** = HSM dedicado (FIPS L3).
- ✅ **ACM** emite certificados **SSL/TLS gratuitos**.
- ✅ **Secrets Manager** = rotação automática; **Parameter Store** = grátis, sem rotação.
- ✅ Criptografia em trânsito = TLS; em repouso = KMS/SSE.

---

## Quadro "Quando usar?"

| Situação / Necessidade | Serviço | Por quê |
|---|---|---|
| Criptografar bucket S3, volume EBS, snapshot RDS | **KMS** | Integração nativa com serviços AWS, baixo custo |
| Compliance bancário / governamental (FIPS 140-2 **Level 3**) | **CloudHSM** | HSM dedicado, AWS não tem acesso às chaves |
| Controlar 100% do ciclo de vida da chave (sem AWS no meio) | **CloudHSM** | Single-tenant, isolamento total |
| Rotacionar chave automaticamente a cada 1 ano | **KMS** (Customer Managed) | Rotação automática nativa |
| Certificado SSL/TLS **gratuito** para CloudFront / ELB / API Gateway | **ACM** | Emissão e renovação automáticas sem custo |
| Certificado SSL para servidor **fora da AWS** ou EC2 direto | ❌ Não use ACM público | ACM não exporta chave privada (use ACM Private CA ou terceiros) |
| Armazenar **senha de banco RDS** com rotação a cada 30 dias | **Secrets Manager** | Rotação automática integrada com RDS |
| Armazenar **API key de terceiros** (Stripe, SendGrid) | **Secrets Manager** | Segredo sensível + auditoria |
| Armazenar **endpoint, URL ou flag de feature** | **Parameter Store** | Grátis no tier padrão |
| Armazenar string de configuração criptografada **sem custo** | **Parameter Store** (SecureString) | Criptografia via KMS, sem cobrança por segredo |
| Criptografia **em trânsito** (HTTPS, dados na rede) | **TLS via ACM** | Certificado no Load Balancer / CloudFront |
| Criptografia **em repouso** (dados no disco) | **KMS** (SSE-KMS) | Padrão para S3, EBS, RDS |

### Regra de bolso para o exame

> 🔑 **Chave** → KMS (padrão) ou CloudHSM (FIPS L3)
> 📜 **Certificado** → ACM
> 🔐 **Segredo com rotação** → Secrets Manager
> ⚙️ **Config grátis** → Parameter Store
