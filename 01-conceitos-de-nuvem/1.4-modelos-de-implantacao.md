# 1.4 — Modelos de Implantação

## Os 3 Modelos Principais

### 1. Nuvem Pública
- Recursos compartilhados via internet.
- **Exemplo**: AWS, Azure, GCP.
- **Vantagens**: sem CapEx, elástico, global.

### 2. Nuvem Privada (On-Premises)
- Infra dedicada a uma única organização.
- **Vantagens**: controle total, conformidade específica.
- **Desvantagens**: CapEx alto, manutenção interna.

### 3. Nuvem Híbrida
- Combinação de **pública + privada/on-premises**.
- **Exemplos AWS**: **AWS Outposts**, **Storage Gateway**, **Direct Connect**.
- **Uso típico**: dados sensíveis on-premises, workloads elásticos na nuvem.

---

## Comparação

| Aspecto | Pública | Privada | Híbrida |
|---------|---------|---------|---------|
| Custo inicial | Baixo | Alto | Médio |
| Escalabilidade | Alta | Limitada | Alta |
| Controle | Médio | Total | Variável |
| Conformidade | Depende | Alta | Alta |

---

## Serviços AWS para Cenários Híbridos

- **AWS Outposts** — hardware AWS no seu data center.
- **AWS Storage Gateway** — integra storage on-prem com S3/EBS/FSx.
- **AWS Direct Connect** — link dedicado para AWS.
- **VMware Cloud on AWS** — VMware nativo na AWS.

---

## Pontos-Chave para o Exame

- ✅ **Híbrida** é a combinação de pelo menos **pública + privada**.
- ✅ **Outposts** é a resposta clássica para "AWS no meu data center".
- ✅ Nuvem Pública ≠ inseguro — segurança é **responsabilidade compartilhada**.
